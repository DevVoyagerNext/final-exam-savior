# 软件需求规格说明书 V1

- 项目名称：`期末救星`
- 文档目标：将当前产品想法整理为可开发、可验收的正式需求基线
- 产品形态：`Web` 系统
- 核心用户：大学生个人用户
- 核心能力：管理员上传学习资料，自动生成题目、知识点和扩展题，并支持在线预览、在线复习与下载源文件及 HTML

## 一、产品目标

- 将“死板”的课件、讲义、笔记转化为“动态”的复习内容，提升期末复习效率。
- 降低用户手工提炼知识点、手工整理习题、手工归档资料的时间成本。
- 建立“上传资料 -> 异步生成 -> 结果通知 -> 在线预览/在线学习/下载”的完整闭环。
- 在第一版中实现可用、可演示、可持续扩展的最小可行产品。

## 二、用户角色

### 1. 普通用户

- 登录后可查看有权限的文件列表。
- 可查看文件详情、在线预览原文件、下载原文件、在线预览生成结果、在线练习、背诵知识点。
- 不可上传文件，不可删除文件，不可管理分类，不可管理邀请码。

### 2. 管理员

- 拥有普通用户全部权限。
- 可增删改查文件分类。
- 可上传和删除文件。
- 可设置上传文件为 `公开` 或 `仅自己可见`。
- 可生成并管理邀请码。
- 可查看系统中的文件与生成记录。

### 3. 未登录用户

- 不可浏览文件，不可查看生成内容，不可下载文件。

## 三、产品范围

- `账号与权限模块`
- `邮箱验证码模块`
- `极验验证模块`
- `登录安全风控模块`
- `邀请码注册模块`
- `文件分类模块`
- `文件上传与存储模块`
- `异步生成任务模块`
- `题目/知识点/扩展题生成模块`
- `结果查看、在线预览与下载模块`
- `文件检索与筛选模块`
- `通知模块`

## 四、业务流程

- 用户使用 `QQ 邮箱` 作为账号进行注册。
- 用户注册前先完成极验滑动验证。
- 用户请求发送邮箱验证码，系统校验发送频率后向邮箱发送验证码。
- 用户填写邮箱验证码、邀请码和密码后完成注册。
- 注册成功后，系统应直接完成登录并返回登录态。
- 用户登录时先完成极验滑动验证，再进行账号密码校验。
- 用户进入文件列表页，查看自己可访问的文件。
- 用户忘记密码时，先完成极验滑动验证，再通过邮箱验证码重置密码。
- 管理员先选择文件分类和文件可见范围，再上传源文件。
- 系统计算文件哈希值，判断是否已有相同内容文件。
- 若哈希已存在，则不重复生成，并按最新上传信息覆盖旧记录。
- 当哈希已存在且复用旧结果时，系统仍需创建一条任务记录用于展示，并将该任务直接标记为成功。
- 若哈希不存在，则将源文件上传至对象存储，并创建一条生成记录和异步总任务。
- 若源文件为图片扫描件，系统先通过 OCR 提取文字内容，再交由大模型 API 生成结果。
- 系统异步并行生成 3 类 HTML：题目、知识点、扩展题。
- 每个 HTML 生成完成后立即上传到对象存储，并更新对应子任务状态和 HTML URL。
- 系统向用户发送任务完成通知。
- 用户可在线预览源文件和生成结果，也可下载源文件与 HTML 文件离线使用。

## 五、功能需求

### 5.1 用户注册

- 用户账号唯一标识为邮箱地址。
- 第一版注册邮箱仅支持 `QQ 邮箱`。
- 同一个 `QQ 邮箱` 仅允许注册一个账号。
- 用户注册时必须填写 `QQ 邮箱`、邮箱验证码、密码、确认密码、邀请码。
- 用户发起注册验证码发送前，必须先完成极验滑动验证。
- 系统校验邮箱格式及邮箱域名是否为 `qq.com`。
- 系统校验邀请码是否存在且剩余可用次数大于 0。
- 系统校验邮箱验证码是否正确、是否过期、是否已失效。
- 校验通过后创建用户账号，并将邀请码剩余次数减 1。
- 当邀请码剩余次数减为 0 时，系统自动删除该邀请码记录。
- 注册成功后，系统应直接为用户创建登录态，无需用户再次手动登录。
- 若邀请码不存在或剩余次数不足，则注册失败。
- 若邮箱已注册，则注册失败。

### 5.2 邮箱验证码

- 邮箱验证码适用于注册和忘记密码场景。
- 发送验证码前必须先完成极验滑动验证。
- 验证码需绑定邮箱地址、业务场景和失效时间。
- 邮箱验证码有效期为 `3 分钟`。
- 验证码发送频控第一版仅按邮箱维度统计。
- 同一邮箱发送频率限制如下：
- `60 秒内最多发送 1 次`
- `5 分钟内最多发送 3 次`
- `3 小时内最多发送 5 次`
- `24 小时内最多发送 7 次`
- 单个验证码最多允许输错 3 次，达到 3 次后立即失效。
- 验证码验证成功后立即失效，不可重复使用。

### 5.3 用户登录

- 用户使用邮箱和密码登录。
- 登录前必须先完成极验滑动验证。
- 登录成功后返回用户身份、角色信息和有效期为 `7 天` 的登录态。
- 未登录用户访问受保护页面时，系统应跳转至登录页。
- 登录密码规则为 `至少 8 位，允许任意字符`。
- 密码必须采用强哈希加盐方式存储。
- 登录失败风控第一版仅按邮箱维度统计。
- 登录密码错误风控规则如下：
- `1 小时内失败 3 次`，禁止登录 1 小时。
- `3 小时内失败 5 次`，禁止登录 3 小时。
- `24 小时内失败 7 次`，禁止登录 24 小时。
- 当同时命中多个时间窗口限制时，系统按 `最长封禁时长` 执行。
- 用户需支持主动退出当前登录态。
- 若账号状态为禁用，则用户不可登录。

### 5.4 修改密码

- 已登录用户可通过输入旧密码、新密码和确认密码修改密码。
- 修改密码成功后，当前用户的全部旧登录态应立即失效，并要求重新登录。
- 新密码必须符合系统密码策略。

### 5.5 忘记密码

- 用户可通过邮箱验证码重置密码。
- 发起忘记密码验证码发送前，必须先完成极验滑动验证。
- 用户输入邮箱、邮箱验证码、新密码、确认密码后完成密码重置。
- 新密码不得与系统密码策略冲突。
- 密码重置成功后，当前验证码立即失效。
- 密码重置成功后，该用户全部旧登录态应立即失效。

### 5.6 极验验证

- 极验滑动验证适用于以下场景：
- `注册前发送邮箱验证码`
- `忘记密码前发送邮箱验证码`
- `登录提交前`
- 极验校验失败时，不允许继续发码、注册、登录或重置密码，并提示用户重新验证。

### 5.7 账号状态管理

- 账号状态第一版至少包括：`启用`、`禁用`。
- 管理员可在后台禁用或启用用户账号。
- 用户账号被禁用后，不可继续登录。
- 用户账号被禁用时，当前全部登录态应立即失效。
- 系统初始化管理员账号时，应通过环境变量注入管理员邮箱和初始密码。

### 5.8 邀请码管理

- 仅管理员可创建邀请码。
- 邀请码永不过期。
- 邀请码支持单条创建和批量生成。
- 批量生成时，同一批邀请码应共享相同配置。
- 单次批量生成数量上限为 `100` 个。
- 管理员创建邀请码时必须设置可使用次数。
- 管理员创建邀请码时支持填写备注信息。
- 邀请码支持系统随机生成，也支持管理员自定义。
- 自定义邀请码必须全局唯一，若重复则创建失败。
- 邀请码仅在用户注册成功后扣减 1 次。
- 当邀请码剩余次数减为 0 时，系统自动从数据库删除该邀请码记录。
- 管理员可查看邀请码列表及其初始次数、剩余次数和备注。
- 管理员可修改邀请码备注，但不可修改邀请码内容和可使用次数。
- 管理员可手动停用邀请码，停用后直接从数据库删除。
- 第一版不保留邀请码使用记录。
- 批量生成完成后，管理员可在页面直接查看邀请码结果，也可导出 `xlsx` 文件。

### 5.9 文件分类管理

- 系统初始化存在一个默认分类。
- 默认分类不可删除、不可修改名称、不可禁用。
- 管理员可新增、编辑、删除其他分类。
- 若分类下仍存在文件，则禁止删除该分类。
- 普通用户仅可查看分类列表。
- 上传文件时必须选择一个分类。

### 5.10 文件上传

- 第一版仅管理员可上传文件并触发 AI 生成。
- 上传时必须填写或选择以下信息：文件、分类、可见范围。
- 可见范围至少包含：
- `公开`：所有登录用户可查看。
- `仅自己可见`：仅上传该文件的管理员本人可查看。
- 支持文件类型：`PPT/PDF`、`Word`、`图片扫描件`、`纯文本/Markdown`。
- 系统上传前或上传后必须计算文件哈希值。
- 若上传文件为 `Word/PPT`，系统需支持后续在线预览时转换为 `PDF`。
- 若文件哈希相同，则系统不重复生成内容，并以最新上传信息覆盖旧记录。
- 若文件哈希相同且复用旧结果，系统仍需创建一条用于展示的任务记录，并将任务状态直接标记为成功。

### 5.11 文件权限控制

- 普通用户可查看：
- 所有公开文件。
- 管理员可查看全部文件。
- 普通用户不可查看管理员设为 `仅自己可见` 的文件。

### 5.12 文件查询

- 支持按分类查看文件列表。
- 支持按文件名模糊搜索。
- 支持按可见范围筛选文件。
- 文件列表需展示基础信息：文件名、分类、上传者、上传时间、可见范围、生成状态。
- 文件列表和文件详情页需提供在线预览、查看生成结果和下载入口。

### 5.13 文件删除

- 第一版仅管理员可删除文件。
- 文件删除采用直接物理删除方式。
- 管理员删除文件前，系统必须展示删除影响说明，并要求输入确认后方可执行删除。
- 删除文件时，需同时删除对象存储中的源文件、3 个 HTML 结果文件以及预览 PDF 文件。
- 文件删除不影响既有任务记录和通知记录的保留。
- 若用户通过历史任务或历史通知访问已删除文件，系统应提示资源不存在。
- 第一版不保留文件删除日志。
- 若任一对象存储资源删除失败，则数据库删除操作必须整体回滚，保证数据一致性。

### 5.14 异步任务生成

- 上传成功后，系统创建一条生成记录和一条异步总任务。
- 三个 HTML 必须采用异步并行方式生成，而非串行生成。
- 总任务状态至少包括：`待处理`、`处理中`、`部分成功`、`成功`、`失败`。
- 子任务状态至少包括：`待处理`、`处理中`、`成功`、`失败`。
- 任务执行由后台任务队列消费。
- 当 HTML 生成子任务或预览 PDF 转换任务失败时，系统应自动重试 `3` 次。
- 生成完成后通知用户查看结果。
- 生成失败后通知用户失败原因或提示重试。
- 当部分 HTML 生成成功时，系统允许用户先查看和下载已成功结果。
- 当上传文件哈希重复且直接复用旧结果时，系统不重新进入生成队列，但仍需创建一条成功任务记录。
- 管理员手动重试时，仅允许重试失败子任务，不支持整条任务重跑。
- 管理员手动重试次数第一版不设上限。
- 任务记录保留期为 `30 天`，到期后由系统自动清理。

### 5.15 大模型生成

- 系统通过接入大模型 API 完成内容生成。
- 图片扫描件需先经 OCR 提取文字，再进入大模型生成流程。
- 题目与知识点生成应优先依据源文件原始标题结构划分章节；若原始结构不清晰，则由 AI 自动补充分组。
- 基础题和扩展题数量应根据源文件中的知识点数量和原始题目数量动态生成。
- 每个知识点至少生成 `1` 道基础题。
- 若源文件中本身包含题目，系统应至少保留并生成这些原题。
- 当某一章节内容过少时，系统应自动合并到相邻章节后再生成题目和知识点。
- 生成内容整体以中文为主，必要专业术语可保留英文。
- 生成内容包括：
- `题目 HTML`
- `知识点 HTML`
- `扩展题 HTML`
- 第一版采用“直接生成可用”策略，不设置人工审核流。

### 5.16 题目 HTML

- 支持单选题、多选题、判断题、填空题。
- 页面应采用统一的卡片式视觉风格，整体风格参考已确认示意图。
- 页面顶部应使用固定模板头图，展示页面标题、副标题和标签。
- 页面应支持用户作答，并在用户选择后立即判定对错。
- 页面应展示正确答案。
- 页面应展示正确答案解析、错误答案解析、知识点总结和难度标签。
- 解析展示顺序为：`判定结果 -> 正确答案解析 -> 错误答案解析`。
- 正确答案解析和错误答案解析采用简明解析风格，便于快速复习。
- 错误答案解析仅展示用户当前选错项的解析内容。
- 知识点总结应以 `1-3 条`简要总结形式呈现。
- 题目内容应按章节组织展示。
- 填空题判题需忽略大小写差异和多余空格。
- 多选题必须全选正确选项且不能多选、少选，才判定为正确。
- 页面应支持重复作答。
- 页面应兼容桌面端和手机浏览器。
- 题目页、知识点页、扩展题页之间应支持页内标签切换。
- 题目页应展示答题进度信息。
- 下载后的 HTML 应尽量内嵌样式和脚本，支持单文件离线打开。

### 5.17 知识点 HTML

- 页面应展示从源文件提炼出的核心知识点。
- 页面用于背诵与记忆。
- 页面应采用与题目页统一的视觉风格和固定模板头图。
- 页面应采用 `大纲折叠` 方式展示知识点内容。
- 页面应提供说明区，用于展示详细解释、记忆提示或易错提醒。
- 页面应兼容桌面端和手机浏览器。
- 下载后的 HTML 应尽量内嵌样式和脚本，支持单文件离线打开。

### 5.18 扩展题 HTML

- 页面应基于源内容生成进阶或补充练习。
- 页面应采用与题目页统一的视觉风格和固定模板头图。
- 页面交互能力、解析内容和离线交付能力与题目 HTML 保持一致。
- 扩展题难度应较基础题略高一档，但不应明显脱离原课程复习范围。

### 5.19 结果查看、在线预览与下载

- 用户可在线预览生成结果。
- 用户可在线预览源文件。
- 用户可下载生成的独立 HTML 文件。
- 用户可下载源文件。
- 在线预览和下载前均需校验文件权限。
- 生成的题目页、知识点页、扩展题页在线预览时，应直接打开对应结果页。
- 第一版源文件在线预览需支持 `PDF`、`图片`、`TXT/Markdown`、`Word`、`PPT`。
- `Word/PPT` 文件在首次预览时转换为 `PDF`，转换成功后需保存到对象存储并供后续预览复用。
- 当 `Word/PPT` 转 `PDF` 尚未完成时，预览页需同时展示处理中状态和原文件下载入口。
- 当 `Word/PPT` 转 `PDF` 失败时，管理员可手动重新触发转换。
- `TXT/Markdown` 在线预览应采用排版渲染方式展示内容。
- 当 `PDF`、`TXT/Markdown` 文件体积较大时，系统应优先保证阅读体验，可采用懒加载或分页加载策略。
- 多页扫描图片或多张图片在线预览时，应采用纵向连续展示方式。
- PDF 预览应采用整页滚动方式浏览。
- 预览页应提供文件基础信息、下载按钮和生成结果入口。
- 预览入口需同时存在于文件列表页、文件详情页和我的任务页。
- 桌面端源文件预览页采用左侧主预览区、右侧信息操作区布局。
- 手机端源文件预览页采用上方主预览区、下方信息操作区布局。
- 源文件和 HTML 的下载、预览访问均需先经过后端鉴权，再返回可访问资源链接。
- 后端鉴权通过后，预览和下载接口应返回有效期为 `10 分钟` 的临时资源 URL。
- 私有文件在线预览时，每次请求均需经过后端严格鉴权。

### 5.20 结果复用

- 对于相同哈希的文件，系统不再次生成。
- 系统应按最新上传信息覆盖旧记录。
- 数据库只保留最近一次生成记录。
- 一个记录包含：
- `uploadUserId`
- `sourceFileUrl`
- `questionHtmlUrl`
- `knowledgeHtmlUrl`
- `extendedQuestionHtmlUrl`
- `uploadTime`
- `sourceFileName`
- `sourceFileType`
- `sourceFileSize`
- `sourceFileHash`
- `totalTaskStatus`
- `questionTaskStatus`
- `knowledgeTaskStatus`
- `extendedQuestionTaskStatus`

### 5.21 通知

- 第一版通知模块采用站内通知方式。
- 任务成功、任务部分成功、任务失败时，系统应向上传该文件的管理员发送站内通知。
- `Word/PPT` 转 `PDF` 预览转换成功或失败时，系统应向上传该文件的管理员发送站内通知。
- 通知类型至少包括：`生成成功`、`部分成功`、`生成失败`、`预览转换成功`、`预览转换失败`。
- 通知列表应展示通知标题和摘要内容。
- 通知内容应包含通知标题、通知类型、简要内容、创建时间、已读状态和跳转目标。
- 任务失败或预览转换失败的通知应展示简要失败原因，并提供查看详情入口。
- 通知支持 `已读/未读` 状态。
- 用户点击通知进入详情后应自动标记为已读。
- 用户可手动将单条或多条通知标记为已读。
- 同一任务多次重试失败时，连续失败通知应合并处理，避免重复堆积。
- 第一版需要独立的通知详情页。
- 页面顶部未读通知数量应显示精确数字。
- 通知应保留 `30 天`，超过保留期后在用户访问通知模块时进行懒清理。
- 用户第一版不可手动删除通知。
- 点击通知后应进入通知详情页，并提供跳转至对应任务详情页的入口。
- 页面顶部需展示未读通知数量。
- 第一版可先支持站内通知。
- 后续可扩展邮件或短信通知。

## 六、非功能需求

### 6.1 性能

- 普通页面操作响应时间应尽量控制在 2 秒内。
- 文件上传应提供进度反馈。
- 生成任务采用异步机制，避免阻塞前端请求。
- 三个 HTML 生成任务应支持并行执行，减少整体等待时间。
- `Word/PPT` 转 `PDF` 应按首次预览触发的异步方式执行，避免阻塞上传链路。

### 6.2 可扩展性

- 后续可扩展更多文件格式、更多题型、更多通知方式。
- 后续可接入多模型切换机制。

### 6.3 安全性

- 所有受保护资源必须在登录后访问。
- 文件访问必须校验可见性和角色权限。
- 对象存储 URL 建议采用受控访问策略，避免私有文件被直接暴露。
- 密码必须采用强哈希加盐方式存储。
- 邮箱验证码必须加密或脱敏存储，并具备过期与错误次数控制。
- 登录风控和发码频控第一版仅按邮箱维度统计，并保留后续扩展到 IP 等维度的能力。
- 密码重置、修改密码、账号禁用等安全操作发生后，相关旧登录态必须立即失效。

### 6.4 可维护性

- 任务状态、错误日志、模型调用日志需要可追踪。
- 文件、生成记录、通知记录应具备清晰的数据关联。
- 预览转换状态和转换失败原因应具备可追踪能力。
- 自动重试次数、手动重试记录和任务保留清理行为应具备可追踪能力。
- 通知发送结果、已读状态和自动清理行为应具备可追踪能力。

### 6.5 可用性

- 上传失败、生成失败、权限不足、邀请码失效等场景应给出明确提示。
- 在线预览失败或格式暂不支持预览时，应给出明确下载引导。
- 资源已删除或预览资源不存在时，应给出明确提示。

### 6.6 兼容性

- 生成的 HTML 页面应兼容主流桌面浏览器和手机浏览器。

## 七、对象存储与数据规则

- 源文件必须先上传至对象存储。
- 生成出的 3 个 HTML 文件也必须上传至对象存储。
- `Word/PPT` 首次预览转换得到的 `PDF` 也必须上传至对象存储。
- 数据库中仅保存对象存储 URL，不保存大文件本体。
- 数据库仅保留最近一次生成记录。
- 若文件内容哈希相同，则视为同一源文件内容，不再重复生成。
- 若文件名不同但哈希相同，也按重复文件处理，并覆盖旧记录中的对应信息。
- 管理员删除文件时，数据库记录和对象存储中的源文件、HTML 文件、预览 PDF 均需同步物理删除。

## 八、异常场景

- 邮箱格式非法或非 `QQ 邮箱`：注册失败，并提示当前版本仅支持 `QQ 邮箱`。
- 同一邮箱重复注册：注册失败，并提示该邮箱已注册。
- 邀请码不存在：注册失败，并提示邀请码无效。
- 邀请码剩余次数不足：注册失败，并提示邀请码已失效。
- 极验校验失败：禁止发送验证码、登录或重置密码，并提示用户重新验证。
- 邮箱验证码发送过于频繁：发送失败，并提示稍后再试。
- 邮箱验证码错误 3 次：当前验证码立即失效，需要重新获取。
- 登录触发风控限制：禁止登录，并提示剩余限制时间。
- 忘记密码验证码错误、过期或已失效：密码重置失败。
- `Word/PPT` 转 `PDF` 处理中：提示用户稍后刷新，并提供原文件下载入口。
- `Word/PPT` 转 `PDF` 失败：提示管理员可重新触发转换。
- 历史任务或历史通知对应的文件已被删除：提示资源不存在。
- 文件删除过程中任一资源删除失败：回滚删除结果，并提示删除失败。
- 未选择分类：上传失败，并提示必须先选择分类。
- 文件解析失败：任务失败，通知用户重新上传或更换格式。
- OCR 识别失败：任务失败，并提示用户更换更清晰的扫描件。
- 大模型 API 调用失败：任务失败，记录失败原因。
- 对象存储上传失败：任务失败，记录失败原因。
- 用户访问无权限文件：返回无权限提示。
- 用户预览私有文件但无权限：拒绝预览。
- 用户下载私有文件但无权限：拒绝下载。
- 哈希重复文件上传：不重新生成，并覆盖旧记录后提示用户。
- 账号被禁用：立即使当前登录态失效，并提示账号已被禁用。

- 普通用户端与管理员端主要页面均需完整适配手机端。
- 注册、登录、上传、重试等提交操作应以页内提示和状态文案为主要反馈方式。

## 十一、验收标准

- 用户必须使用有效 `QQ 邮箱`、有效邀请码和有效邮箱验证码完成注册。
- 同一个 `QQ 邮箱` 只能注册一个账号。
- 邮箱验证码有效期为 `3 分钟`。
- 邀请码由管理员创建时配置可使用次数和备注，且永不过期。
- 邀请码支持随机生成和自定义，自定义邀请码必须唯一。
- 邀请码支持批量生成，且单次批量生成上限为 `100` 个。
- 注册成功后邀请码剩余次数减 1，剩余次数为 0 时自动从数据库删除。
- 注册成功后系统必须直接为用户创建登录态，无需再次手动登录。
- 管理员仅可修改邀请码备注，不可修改邀请码内容和可使用次数。
- 管理员可手动停用邀请码，停用后直接删除。
- 第一版不保留邀请码使用记录。
- 批量生成邀请码结果必须支持导出为 `xlsx` 文件。
- 注册和忘记密码发送验证码前必须完成极验滑动验证。
- 登录时必须完成极验滑动验证。
- 极验验证失败时，系统必须明确提示用户重新验证。
- 同一邮箱验证码发送必须满足 `60 秒 1 次、5 分钟 3 次、3 小时 5 次、24 小时 7 次` 的限制。
- 单个验证码输错 3 次后必须立即失效。
- 邮箱验证码错误次数必须按单个验证码独立统计。
- 忘记密码可通过邮箱验证码完成密码重置。
- 密码必须采用强哈希加盐方式存储。
- 登录失败风控和验证码发送频控第一版仅按邮箱维度统计。
- 登录成功后必须生成有效期 `7 天` 的登录态。
- 系统必须支持用户主动退出登录。
- 已登录用户必须支持通过旧密码修改密码。
- 修改密码成功后，该用户全部旧登录态必须立即失效并重新登录。
- 通过邮箱验证码重置密码成功后，该用户全部旧登录态必须立即失效。
- 系统必须支持账号启用/禁用状态管理。
- 账号被禁用后，用户不可登录，且全部已有登录态必须立即失效。
- 管理员账号必须通过环境变量注入方式初始化。
- 未登录用户无法查看文件列表和文件详情。
- 登录失败达到 `1 小时 3 次、3 小时 5 次、24 小时 7 次` 时，系统必须按最长封禁规则限制登录。
- 第一版仅管理员可上传文件并触发 AI 生成。
- 普通用户仅可查看公开文件，不能查看管理员私有文件。
- 管理员可管理分类，默认分类不可编辑删除。
- 分类下存在文件时，系统必须禁止删除该分类。
- 上传文件时必须选择分类，否则上传失败。
- 上传后必须进入异步任务队列，而不是同步阻塞等待。
- 三个 HTML 必须并行异步生成，并记录总任务状态和子任务状态。
- HTML 生成失败或预览 PDF 转换失败后，系统必须自动重试 `3` 次。
- 当部分 HTML 生成成功时，系统允许先预览和下载成功结果。
- 当某个 HTML 子任务失败时，管理员只能单独手动重试失败子任务，且手动重试次数不受限制。
- 任务记录保留期为 `30 天`，到期后系统自动清理。
- 文件删除仅允许管理员执行，且必须输入确认后才可删除。
- 文件删除时必须同步删除源文件、3 个 HTML 结果和预览 PDF。
- 若文件删除过程中任一资源删除失败，系统必须整体回滚删除结果。
- 文件删除后，历史任务和通知可保留，但访问相关资源时必须提示资源不存在。
- 任务完成后用户能收到结果通知。
- 第一版站内通知必须覆盖任务成功、部分成功、失败以及预览转换成功、失败场景。
- 通知必须支持已读/未读状态，并支持自动已读与手动/批量已读。
- 通知列表必须展示标题和摘要，并提供独立通知详情页。
- 点击通知后必须进入通知详情页，并可继续跳转到对应任务详情页。
- 页面顶部必须展示精确未读通知数量。
- 通知列表必须支持按已读状态和通知类型筛选。
- 任务失败或预览转换失败通知必须展示简要失败原因和详情入口。
- 同一任务多次连续失败时，失败通知必须合并处理。
- 通知保留期为 `30 天`，到期后在用户访问通知模块时进行懒清理。
- 用户第一版不能手动删除通知。
- 同哈希文件不会重复生成。
- 重复哈希文件上传时，系统不重新生成，并按最新上传信息覆盖旧记录。
- 重复哈希文件上传时，系统仍需创建一条成功任务记录，并标识为复用旧结果。
- 系统能生成并保存 3 个 HTML 文件。
- 用户可在线预览源文件和生成结果，并下载源文件与 HTML。
- 源文件和 HTML 的下载、预览必须先经过后端鉴权，再返回资源链接。
- 预览和下载接口返回的临时资源 URL 有效期必须为 `10 分钟`。
- `Word/PPT` 首次在线预览时必须异步转换为 `PDF`，并将转换结果保存到对象存储。
- `Word/PPT` 转换未完成时，页面必须同时展示处理中状态和下载入口；转换失败时管理员可重新转换。
- PDF 预览采用整页滚动方式；图片采用纵向连续展示；`TXT/Markdown` 采用排版渲染展示。
- 预览页必须展示文件基础信息、下载按钮和生成结果入口。
- 生成结果预览必须直接打开对应结果页。
- 桌面端源文件预览页采用左内容右信息布局，手机端采用上下排布。
- 大体积 `PDF`、`TXT/Markdown` 预览时，系统必须优先保证可读性，可采用懒加载或分页加载。
- 旧任务或旧通知对应资源被删除时，预览页必须提示资源不存在。
- 题目页面可在用户选项后立即判定正误，并支持重复作答。
- 题目页必须包含正确答案解析、错误答案解析、知识点总结和难度标签。
- 错误答案解析仅展示用户选错项，知识点总结以 `1-3 条`简要总结形式呈现。
- 基础题和扩展题必须按内容长度动态生成，并按章节组织；章节识别采用原始标题优先、AI 补充分组策略。
- 基础题和扩展题数量必须根据知识点数量和原始题目数量动态生成。
- 每个知识点至少生成 `1` 道基础题；若源文件本身存在题目，系统必须至少保留并生成这些原题。
- 扩展题难度需较基础题略高一档。
- 填空题判题必须忽略大小写和多余空格。
- 多选题必须全选正确选项且不能多选、少选才判定为正确。
- 知识点页必须采用大纲折叠形式展示，并带说明区。
- 知识点页内容必须优先提炼考试高频点。
- 生成的 HTML 页面必须兼容桌面端和手机端，并支持单文件离线打开。
- 文件列表支持按分类、文件名和可见范围筛选。
- 登录后的默认首页必须为文件列表页。
- 第一版不提供独立仪表盘首页。
- 题目页、知识点页、扩展题页必须支持页内标签切换。
- 题目页必须展示答题进度。
- 任务页状态必须支持自动轮询和手动刷新。
- 第一版我的任务页和相关任务接口仅向管理员开放，普通用户访问时必须返回无权限。
- 管理员后台的文件、用户、邀请码列表必须支持分页。
- 用户管理页必须支持按邮箱搜索，并展示用户注册时间。

## 十二、建议的技术实现方向

- `前端`：`Vue` 或 `React`
- `后端`：`Spring Boot` 或 `Go + Gin/Fiber`
- `数据库`：`MySQL` 或 `PostgreSQL`
- `对象存储`：`MinIO`、阿里云 OSS、腾讯云 COS
- `异步任务队列`：`RabbitMQ`、`Redis Stream`、`Celery` 类方案
- `AI 接入`：统一封装大模型 API 调用层，便于后续切换模型供应商
- `权限控制`：基于角色 + 资源可见范围双重校验
